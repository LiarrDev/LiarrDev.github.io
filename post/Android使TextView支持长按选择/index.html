Android 使 TextView 支持长按选择

我们知道，『WeChat』可以通过双击聊天内容对文本进行选择和复制操作：

但是，在 Android 中 TextView 默认模式下是不允许长按选择的，那么这个操作是如何实现的呢？

网上大多数教程都是自定义一个控件继承自 EditText 来实现，或者直接修改 EditText 的控件属性：

<EditText
    ...
    android:background="@null"
    android:editable="false"
    android:textCursorDrawable="@null" />

目的是使 EditText 取消底部横线、不可编辑以及取消游标，以达到伪装成 TextView 的效果。

宏观来说，XML 是一种可扩展标记语言，用元素和属性来描述数据，而不提供数据的显示方法。意思是，Andorid 中使用 XML 描述其是什么，而不是表现的形式，与 HTML 的作用基本一致。

所以当我们使用 EditText 来伪装 TextView 可以说是违背了这一原则。

那么 TextView 中如何使其可以通过长按对文本进行相应的操作呢？

很简单，一句属性即可：

<TextView
    ...
    android:textIsSelectable="true" />

效果如下：

据说该属性在 API 11 后引入，也就是大约 2011 年左右，但是我发现很多 2014 年后甚至到 2018 年技术博客都在介绍使用 EditText 代替 TextView 或者其他自定义控件的方法，感觉莫名其妙。
